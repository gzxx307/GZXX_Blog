## 项目简介

GZXX_Blog 是一个使用纯前端代码实现的静态网站。

首先里面有我自己的学习笔记、技术文章和个人资料，以前我都是记在飞书上的，但是飞书客户端又卡，缓存占的又多，用网页又不方便即开即用，所以干脆直接开个仓库用markdown写了，看起来还不错。

该项目使用 HTML、CSS、JavaScript 三个最常见的前端网页构建，不需要任何额外的环境配置就能直接运行。而且代码模块化，方便个人修改和定制。

其他人也可以将自己的文章、音乐以及头像图片放在指定的文件夹里（或者自己改文件路径），即插即用。

## 如何使用

如果想要快速将该仓库改造为自己的博客，只需按照以下步骤：

1. clone/fork该仓库到本地/云端
2. 已有的所有文档、图片以及音乐，删除还是保留请随意。
3. 将你的所有文章以markdown的形式保存，并放置在articles文件夹中。（格式见下）
4. 将头像命名为head.jpg并放入data文件夹中。（或profile.js文件中配置的路径）
5. 在profile.js中填入个人信息。（说明见下）
6. 将所有音乐文件（支持mp3、wav、ogg三种常见格式）放入musics文件夹中。
7. 在artists.js中配置艺术家名称。（说明见下）
8. 在index.html中修改网页名以及导航栏标题。
9. 将自己的markdown格式的个人介绍命名为`about_me`放入articles文件夹下
9. 在命令行中运行`.\start_blog.ps1`。
10. 保存并提交即可。之后如果选择使用github托管静态网站请看github官方文档，使用其他服务同样。

其中有一些格式上的要求，在下文中说明：

### 注意事项

**markdown**

markdown格式文件的建议命名方式为全小写+下划线分隔单词

markdown格式需存在一个标题（使用一个`#`），标题下为`Tags`行，你可以在Tags行中添加你的文章标签（用于文章分类与搜索），Tags下为正文，支持标准markdown格式。

例如：

```markdown
# TITLE

Tags: My_Tag1, My_Tag2

## CONTENT

...

```

在markdown格式中进行插图时，你可以将图片文件放入articles中与该文档名相同的文件夹下。

> 如你的md文件名为`my_markdown.md`，则该文件中插图均放在同级目录的`my_markdown`文件夹下

另外，`about_me.md`不需要标签行。

**profile.js**

该文件中的内容均显示在Home页面（即主页）中左上角的欢迎卡片中显示。具体填写信息的方式可以看我已经天下好的内容时怎么填的。

**artists.js**

由于自行放入的音频文件不带“艺术家”元数据，所以如果希望显示歌曲作者，需要手动在artists.js文件中进行配置。其中mp3字段为音频文件路径（格式不一定必须为mp3格式），name字段则为歌曲作者，不同作者由空格隔开。
