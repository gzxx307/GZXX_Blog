## Lecture2 线代

### 向量的点乘

判断向量前与后的信息，结果>0为同方向，<0为反方向

vector_a · vector_b = [x_a,y_a] · [x_b, y_b] = x_a\*x_b + y_a\*y_b

### 向量的叉乘

输入两个向量，输出同时*垂直*两个向量的新向量

a x b = c

四指从a的方向向b的方向握紧，大拇指指向的就是c的方向

**如何判断两个向量的左右关系**

a×b得到结果是和z轴同向，是正的，说明b在a的左侧

进一步可以判断点是否落在某一三角形内部。

假设有一点P与三角形ABC

AB×AP > 0 说明P在AB左侧

BC×BP > 0 说明P在BC左侧

CA×CP > 0 说明P在CA左侧

说明点P落在三角形ABC内部

### 矩阵

第一个矩阵的列数与第二个矩阵的行数相等才能进行相乘，如：

第一个矩阵M行N列

第二个矩阵N行P列

得到M行P列的新矩阵

**快速运算位置为a行b列的元素**

新矩阵的a行b列元素=第一个矩阵a行和第二个矩阵b列做点积运算![mmm](games101/matrix-matrix-multi.avif)

**矩阵的性质**

- AB != BA
- (AB)C == A(BC)
- A(B+C) == AB + AC
- (A+B)C == AC + BC

**单位矩阵乘向量（用于变换）**

[[-1,0],[0,1]] * [x,y] == [-x,y]

**矩阵的转置**

x_i_j -> x_j_i

### 视图 / 相机变换

我们将该变换分为三部分：

1. **Model变换**：对目标物体进行变换
2. **View变换**：对摄像机的角度进行变换
3. **Projection变换**：将相机观察到的部分投影到视图中

#### View变换

该变换需要做到三件事：

1. 决定相机位置e
2. 决定相机看向的方向g
3. 决定相机头朝上的方向t

我们规定相机永远在(0,0,0)，并沿-Z方向看

变化的永远是其他物体

所以我们要先把相机归到原点，分为3步

1. 平移相机位置e至(0,0,0)
2. 将相机朝向的方向g旋转到-Z
3. 旋转相机头朝上的方向t到Y

总变换矩阵拆分为M<sub>view</sub>=R<sub>view</sub>T<sub>view</sub>

第一步平移直接用平移公式（注意变换矩阵的平移量为负数，因为是平移到原点）

第二三步由于将任意向量旋转到轴上比较难写，我们改为先求将轴旋转到任意向量的矩阵，再对矩阵求逆，即可得到任意向量旋转到轴的矩阵。

![vt](games101/view-transformation.png)

最后求出M<sub>view</sub>，将M<sub>view</sub>应用到相机，相机归零。同时也需要将Mview应用到其他所有物体，让物体和相机的相对位置保持不变

##### Perspective Projection透视投影

传统的欧式几何是在同一平面内生效的法则，在不同平面下就会造成近大远小的情况。

我们先将远平面以及远平面到近平面的所有平面缩放到近平面大小，然后做一次正交投影即可。

根据相似三角形定理，取一点(x,y,z,1)做变换，若近平面的对应点的深度为n，那么则首先有：

[x,y,z,1] => [nx/z,ny/z,unknown,1]

此时z轴未知，后面我们会推导出来

将这个点的所有坐标同时乘z得[nx,ny,unknown,z]，得到有某一矩阵M，满足：

M·[x,y,z,1] = [nx,ny,unknown,z]

可推导出矩阵的一部分，如下图

![p->o](games101/persp-ortho.png)

想补全这个矩阵，需要用到两条已知的性质

1. 近平面的点不会发生变化
2. 远平面的点z的值不会发生变化

应用于近平面的矩阵时，我们可以用n取代z，得到近平面的最终矩阵为[nx,ny,n<sup>2</sup>,n]，推出矩阵的形式必为[0,0,A,B]的形式，且**An+B = n<sup>2</sup>**

对于远平面，中心点[0,0,z,1]经矩阵变化后仍然是中心点[0,0,z,1]，得**Az+B = f<sup>2</sup>**

解得A=n+f,B=-nf
