## Lecture2 线代

### 向量的点乘

判断向量前与后的信息，结果>0为同方向，<0为反方向

vector_a · vector_b = [x_a,y_a] · [x_b, y_b] = x_a\*x_b + y_a\*y_b

### 向量的叉乘

输入两个向量，输出同时*垂直*两个向量的新向量

a x b = c

四指从a的方向向b的方向握紧，大拇指指向的就是c的方向

**如何判断两个向量的左右关系**

a×b得到结果是和z轴同向，是正的，说明b在a的左侧

进一步可以判断点是否落在某一三角形内部。

假设有一点P与三角形ABC

AB×AP > 0 说明P在AB左侧

BC×BP > 0 说明P在BC左侧

CA×CP > 0 说明P在CA左侧

说明点P落在三角形ABC内部

## Lecture5 光栅化

在之前的操作时，我们将所有的物体全部缩放到[-1,1]<sup>3</sup>的立方体中，接下来就需要将他画在屏幕上，这就叫光栅化。

屏幕为二维数组，数组元素为像素，如1920*1080。屏幕坐标系的原点为左下角，向右为x轴正方向，向上为y轴正方向，像素的中心为(x+0.5,y+0.5)。

### 三角形

三角形是光栅化过程一个很重要的概念，现代图形学中几乎所有的模型都使用三角形构建。

**为什么光栅化选择了三角形？**

- 三角形是最基本的多边形，没有比三角形边更少的多边形
- 其他多边形都可以拆分为三角形
- 三角形必定在一个平面内
- 容易定义三角形的里外
- 三角形的三个点定义好后，三角形内任意一点都可以通过线性插值来计算得到（如：定义好三个点的颜色，三角形内任意一点的颜色可以通过三个点的颜色来进行插值计算得到）

#### 三角形如何转化为像素？

由于三角形的边是一个连续的函数，而像素是一个个点，所以我们需要将连续的函数“离散化”。

**简单近似采样**

给定一个连续的函数f(x)，当x=1是得到的f(1)就是1的采样。采样就是把一个函数离散化的过程。

只要有一个定义在屏幕空间的函数，那么我们就能算出来不同像素中心的值是多少

给定一个三角形，对像素中心进行采样来判断中心是否落在三角形内。

但是遍历所有像素开销太大，如图，白色那一列的像素根本没有碰到三角形，所以只要遍历蓝色区域就可以了

我们知道三角形三个顶点的坐标，有了Xmin，Ymin，Xmax，Ymax就可以得到蓝色的区域
，这个蓝色区域就叫做包围盒（轴向包围盒/BoundingBox/AABB）

## Lecture6 反走样 / 抗锯齿

我们需要对光产生的信息离散化产生的像素进行采样，形成照片。但是采样会产生一些问题：

- **走样**：由于采样频率不足，导致连续信号在离散化过程中产生失真‌
- **摩尔纹**：产生扭曲的纹理
- **车轮效应**：高速旋转的车轮看起来静止、缓慢移动甚至反向旋转

原因就是信号变化速度太快了，以至于采样的速度跟不上。

**反走样处理方法：采样前模糊**

先模糊再采样

![as](games101/antialiased-sampling.png)

> 不能先采样再模糊！

要理解为什么会产生走样，我们先需要了解以下内容

### 频率分析基础

**频域、时域**

频域和时域是对信号或系统的两种不同的表示方式。

频域表示是通过分析信号的频率分布来表示信号的方式。在频域中，信号被分解为一系列不同频率的分量，每个分量对应着信号中的一种特定的周期性成分。

时域表示是通过直接分析信号在时间上的变化情况来表示信号的方式。在时域中，信号的变化被直接表示为在时间上的变化，而不是在频率上的变化。

**傅里叶级数展开**

指的是任何一个周期性的函数都可以变成一系列正弦/余弦的线性组合+一个常数项

**傅里叶变换**

一个函数f(x)可以通过变换得到另一个函数F(w)，F(w)也可以逆变换为f(x)

我们对五个不同频率的函数进行采样，图下图所示

![dfs](games101/different-functions-sampling.png)

在f1(x)和f2(x)中进行采样，能够大致还原出f1(x)或f2(x)的函数波形，但是从f3(x)开始，还原出的波形由于采样频率的限制，还原出的波形与原函数有着较大的出入，频率越高越明显。

当我们同时对函数1和函数2进行采样，两个截然不同的函数，得到的采样结果却完全相同，这就是**走样**

### 滤波与卷积

#### 滤波

颜色变化越大的部分频率越高，往往代表着边界。滤波就是抹掉一些特定的频率，这就使我们可以通过滤波的方式对高频与低频信息进行过滤，从而找到边界与中心。

例如下面一张图：

![ff-o](games101/frequency-filter-original.png)

**高通滤波**：得到某一部分的边界

![ff-h](games101/frequency-filter-high.png)

**低通滤波**：得到模糊的图像

![ff-l](games101/frequency-filter-low.png)

同理，我们还可以通过同时过滤高频和低频，得到多种图像，控制边界与模糊。

#### 卷积

可以理解为在信号任意位置用周围做加权平均。在图像处理这种离散情况下，可以看作一个滑动窗口在图像中滑动，对每个位置都计算一次周围元素的加权平均。

**卷积定理**

时域的卷积 == 频域的乘积

时域的乘积 == 频域的卷积

例如：假设我们使用一个3x3的卷积滤波器对一个图片进行卷积操作，你可以直接进行卷积，也可以：

1. 先傅里叶变换这幅图，将这幅图变到频域
2. 将卷积滤波器变到频域上
3. 将两者相乘，乘完后得到的频域的结果，将其逆傅里叶变换，变到时域上

![ct](games101/convolution-theorem.png)

### 采样与走样的频率解释

基于上述知识点，采样可以理解为：将原始信号与冲击函数进行相乘

在频域中，采样相当于复制原始信号的频谱，表现为原始信号的频谱被以采样频率为间隔复制粘贴。

左边一列是时域，右边一列是频域

![se](games101/sampling-equals.png)

对于走样的产生，采样越稀疏，频谱复制间隔越小，频谱重叠越严重。而采样越密集，频谱复制间隔越大，越不易发生重叠。

![a](games101/aliasing.png)

### 反走样

核心思想：先对图像做模糊处理，把高频信息切掉，再采样，减少重叠部分

![aa](games101/antialiasing.png)

这样频域图像就不会发生混叠，也就没有走样了

**MSAA（Multi-Sample Anti-Aliasing）多重采样抗锯齿**

将一个像素划分为多个子样本，例如将一个像素分成四个子样本，判断每个子样本中心是否在三角形内，然后再计算覆盖率作为像素颜色值。

需要注意的是：MSAA并不是通过提高分辨率来解决的，而是近似计算覆盖面积，显示的像素个数还是相同的。

**FXAA（Fast Approximate Anti-Aliasing）快速近似抗锯齿**

FXAA本质上不对模型进行处理，而是一种基于图像的后处理方法。

首先得到有锯齿的图像，通过图像匹配找到边界，此时边界是带有锯齿的。然后再将边界替换为无锯齿的边界。速度更快且效果更好。

**TAA（Temporal Anti-Aliasing）时域抗锯齿**

TAA主要对静止场景效果好，因为该方法利用的是时间维度，即复用上一帧的信息，将MSAA的多个样本分布在时间上，而在当前帧不需要引入额外采样。

**Super Resolution 超分辨率**

超分辨率可以实现输入低分辨率图像而输出高分辨率图像，特别用于解决“采样率不够”的问题。

例如**DLSS（Deep Learning Super Sampling）**使用深度学习网络猜测缺失的细节

## Lecture7 深度缓冲 & 着色

### 深度缓冲

在光栅化过程中，我们需要确定哪些东西能够被最终显示在屏幕上，哪些东西被遮挡在后面，也就是**遮挡关系**的问题。

**画家算法**

在光栅化时，先画最远的物体，然后从远处开始逐渐画近的物体，这样进的物体就会覆盖远的物体。

这种方式最符合尝试，但是会导致一个问题：因为该算法是基于三角形整体坐标计算的，当存在多个相互遮挡（循环遮挡）的物体（三角形）时，我们无法判断三角形的远近，无法定义深度关系。

**Z-Buffer**

深度缓存算法在每个像素中记录其看到的最浅深度（离相机最近的深度），在渲染时不光要存渲染结果，还要存深度结果。

算法流程：

1. 首先将所有像素的深度设为无穷
2. 逐个处理每个三角形，进行光栅化
3. 对于每个在光栅化过程中重叠（同时存在两个三角形）的像素：z<sub>current</sub>
    - 计算新三角形在该像素处的深度
    - 将新深度z<sub>new</sub>与当前记录的深度z<sub>current</sub>进行比较
    - 如果z<sub>new</sub> < z<sub>current</sub>
        - 更新深度缓存z<sub>current</sub> = z<sub>new</sub>
        - 更新帧缓存：写入新三角形的颜色
    - 否则：忽略该三角形

### 着色

着色是指对不同物体应用不同材质的过程

着色模型由三部分组成：

1. 漫反射
2. 高光 / 镜面反射
3. 环境光

#### 着色点

表面上需要计算颜色的点称为**着色点**，渲染一个点，我们需要有以下输入：

- 观察方向v
- 点对应三角形的法向量n
- 光线方向l
- 表面参数（颜色、高光度等）

#### 漫反射

同样的光从不同角度照到同一平面上产生的明暗不一样。根据Lambert余弦定律，平面接收到的能量正比于n·l = cos_theta。在计算机中可以表示为公式max(0,n·l)。

又由于点光源需要向四周均匀辐射能量，根据能量守恒定律，距离光源越远，单位面积能量越小。即能量与距离平方成反比I/r<sup>2</sup>

### 高光

当表面光滑时，反射方向接近镜面反射方向时才能看到高光。为了便于计算，我们引入**半程向量h**：观察方向v与入射方向l的角平分线。

h = normalize(v+l)

所以为了知道能否看到高光，布林冯模型只需要知道n和h是否接近即可

如果用视线方向v和高光方向R来判断能否看到高光，就是冯模型，但是计算反射方向的计算量较大。

在计算是否能看到高光时，如果使用余弦值衡量角的大小，那么可看到高光的范围可能会很大，因为角度在很大的情况下余弦值仍然有很大的值。而正常情况下我们认为的高光都是很小很亮的。

在布林冯反射模型中，高光项Specular=k<sub>s</sub>·(n·h)<sup>p</sup>

其中k<sub>s</sub>为镜面反射系数，通常为白色。p为shiness指数，用于控制高光范围，常用范围约100~200之间

### 着色频率

着色频率不同表现不一样

下图从左到右分别为面着色，顶点着色与像素着色

![sf](games101/shading-frequencies.png)

三种着色频率各有各的优点，需要根据情况判断。一般情况下：

|着色频率|着色位置|效果|
|---|---|---|
|Flat Shading|每个三角形|效果粗糙，可见三角形边界|
|Gouraud Shading|每个顶点|插值得到内部颜色|
|Phong Shading|每个像素|效果最细腻|

不同着色频率产生的效果也取决于模型本身，当几何足够复杂时用FlatShading得到的效果也很好。反之，当几何的面数大于像素数量时FlatShading的性能也不会好于PhongShading。

在实际应用中，我们一般选择Phong Shading

### 图形渲染管线 / 实时渲染管线

![gp](games101/graphics-pipeline.png)

总体就是一下几步：

- 顶点处理：将三维空间的点投影在平面上
- 三角形处理：将这些点连接形成三角形
- 光栅化：将三角形离散成为屏幕上的Fragment（未经处理的像素）
- 着色：给像素上色
- 后处理：深度缓冲-处理遮挡关系，MSAA等抗锯齿

### Shader着色器

现代GPU中，这套渲染管线某些部分是可编程的，可以由开发者去定义顶点/像素如何着色，这部分代码就叫Shader。

如果写的是顶点操作，这个shader就叫做VertexShader（顶点着色器）

如果写的是像素操作，这个shader就叫做FragmentShader（片段/片元着色器）/PixelShader（像素着色器）

### TextureMapping纹理映射

当我们在一个物体上显示一张图片时，我们需要在不同位置显示不同的颜色。由于整体公用同一个着色模型，唯一的区别就是漫反射系数K<sub>d</sub>不同，所以有一种方法，可以定义一个物体上任意一点的基本属性。

3D物体的表面其实都是2D的，比如地球仪，将地球仪上的图撕下来，可以平铺成一张2D的图。物体的表面，通过这种方式可以和一张图有一一对应的关系，这张图就叫纹理。将这张图平铺/裁剪/拉伸到任何物体表面，就叫纹理映射。

纹理上的坐标系通常用UV来表示，每个三角形关联一个UV坐标，三角形内部则通过插值来获得UV坐标。通常约定U和V的范围为[0,1]。
